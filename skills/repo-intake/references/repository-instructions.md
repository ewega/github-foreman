# Repository instructions

Foreman decisions should be based on repository evidence.

Look for:

- `.github/copilot-instructions.md`
- `AGENTS.md`
- `README*`
- `CONTRIBUTING*`
- `.github/workflows/*`
- issue templates
- PR templates
- docs near affected files
- test/build manifests

Use this information for:

- coding-agent custom instructions
- review judgment
- CI expectations
- doc update requirements
- branch strategy
- validation commands

Do not assume a Go, Node, Python, or CLI workflow unless the repository evidence supports it.
